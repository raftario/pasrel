listen

  • listen(port: number, hostname?: undefined | string): Promise<void>
    • Defined in src/index.ts:166

    Starts the server

    The main difference between this method and run is that the promise returned by run only resolves once the server closes, while the promise return by this method will resolve as soon as the server starts litening.

    This method never throws, you are responsible for handling server errors when using it.

    Parameters

    • port: number

      Port to listen on

    • Optional hostname: undefined | string

      Address to listen on (defaults to localhost)

    Returns Promise<void>

    Will resolve when the server starts listening
